# 火灾热释放速率预测深度学习模型技术文档

## 概述

本文档详细介绍了 `fire_hrr_deep_learning_improved.py` 中实现的四种深度学习模型，用于小样本条件下的火灾热释放速率(HRR)曲线预测任务。

---

## 一、ResidualMLP（残差多层感知机）

### 1.1 模型架构

```
输入层 → 线性投影 → [残差块×3] → 输出层
```

### 1.2 核心技术

#### 残差连接 (Residual Connection)
- **原理**: 将输入直接加到输出上，即 `output = input + F(input)`
- **优势**: 
  - 缓解深层网络的梯度消失问题
  - 允许信息直接跨层传递
  - 使网络更容易优化

#### 残差块结构
```
x → Linear → LayerNorm → GELU → Dropout → Linear → LayerNorm → Dropout → (+x) → output
```

#### LayerNorm（层归一化）
- 对每个样本的特征维度进行归一化
- 相比BatchNorm，不依赖batch size，更适合小样本场景

#### GELU激活函数
- 公式: `GELU(x) = x × Φ(x)`，其中Φ是标准正态分布的累积分布函数
- 相比ReLU更平滑，在Transformer等模型中广泛使用

### 1.3 参数规模
- 可训练参数: ~102,145
- 隐藏维度: 128
- 残差块数量: 3

### 1.4 适用场景
- 逐点预测任务
- 学习输入特征到输出的非线性映射
- 计算效率高，适合快速迭代

---

## 二、CNN-BiLSTM（卷积-双向长短期记忆网络）

### 2.1 模型架构

```
输入 → 1D卷积层×2 → 双向LSTM → 输出层
```

### 2.2 核心技术

#### 1D卷积神经网络 (1D-CNN)
- **作用**: 提取序列中的局部特征和模式
- **结构**: 
  ```
  Conv1d(kernel=3) → BatchNorm → GELU → Dropout → Conv1d(kernel=3) → BatchNorm → GELU → Dropout
  ```
- **优势**: 
  - 捕捉时序数据中的局部模式（如峰值、趋势变化）
  - 参数共享，计算效率高
  - 平移不变性

#### 双向LSTM (Bidirectional LSTM)
- **原理**: 同时从前向后和从后向前处理序列
- **结构**:
  - 前向LSTM: 捕捉历史信息
  - 后向LSTM: 捕捉未来信息
  - 输出拼接两个方向的隐藏状态
- **LSTM门控机制**:
  - 遗忘门: 决定丢弃哪些信息
  - 输入门: 决定存储哪些新信息
  - 输出门: 决定输出哪些信息

#### 混合架构优势
1. CNN先提取局部特征，降低序列复杂度
2. BiLSTM再建模长程依赖关系
3. 结合两者优势，既能捕捉局部模式又能理解全局时序

### 2.3 参数规模
- 可训练参数: ~64,449
- CNN隐藏维度: 64
- LSTM层数: 2

### 2.4 适用场景
- 序列到序列预测
- 需要同时考虑局部特征和时序依赖的任务
- 火灾HRR曲线的形状特征提取

## 三、AttentionLSTM（注意力增强LSTM）

### 3.1 模型架构

```
输入 → 线性投影 → 双向LSTM → 自注意力层 → 输出层
```

### 3.2 核心技术

#### 自注意力机制 (Self-Attention)
- **原理**: 让序列中的每个位置都能关注到所有其他位置
- **计算过程**:
  ```
  Attention(Q, K, V) = softmax(QK^T / √d_k) × V
  ```
  其中 Q(Query)、K(Key)、V(Value) 都来自同一输入

#### 多头注意力 (Multi-Head Attention)
- **原理**: 将注意力分成多个"头"并行计算
- **公式**:
  ```
  MultiHead(Q,K,V) = Concat(head_1, ..., head_h) × W^O
  head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
  ```
- **优势**: 
  - 每个头可以关注不同的特征子空间
  - 增强模型的表达能力

#### 残差连接 + LayerNorm
```
output = LayerNorm(x + Dropout(Attention(x)))
```

### 3.3 LSTM + Attention 结合的优势
1. **LSTM**: 捕捉时序依赖，理解序列的演变规律
2. **Attention**: 捕捉全局依赖，识别远距离时间点之间的关系
3. **互补**: LSTM侧重顺序建模，Attention侧重关系建模

### 3.4 参数规模
- 可训练参数: ~241,089
- 隐藏维度: 64
- 注意力头数: 4
- LSTM层数: 2

### 3.5 适用场景
- 需要同时理解时序演变和全局关系的任务
- 序列中存在远距离依赖的情况
- 火灾过程中不同阶段之间的关联分析

## 四、ImprovedTransformer（改进的Transformer）

### 4.1 模型架构

```
输入 → 线性投影 → 位置编码 → LayerNorm → Transformer编码器×2 → 输出层
```

### 4.2 核心技术

#### 相对位置编码 (Positional Encoding)
- **原理**: 为序列中每个位置添加位置信息
- **公式**:
  ```
  PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
  ```
- **作用**: 让模型理解序列中元素的顺序关系

#### Pre-LayerNorm结构
- **传统Post-LN**: `x + Sublayer(LayerNorm(x))`
- **改进Pre-LN**: `LayerNorm(x + Sublayer(x))`
- **优势**: 
  - 梯度更稳定
  - 训练更容易收敛
  - 特别适合深层Transformer

#### Transformer编码器层
```
输入 → LayerNorm → Multi-Head Attention → Dropout → (+残差) 
    → LayerNorm → Feed-Forward → Dropout → (+残差) → 输出
```

#### 前馈网络 (Feed-Forward Network)
```
FFN(x) = GELU(xW_1 + b_1)W_2 + b_2
```
- 隐藏维度: d_model × 4
- 激活函数: GELU

### 4.3 Transformer的优势
1. **并行计算**: 不像RNN需要顺序处理，可以并行处理整个序列
2. **全局感受野**: 每个位置都能直接关注到所有其他位置
3. **灵活的依赖建模**: 通过注意力权重动态决定关注哪些位置

### 4.4 针对小样本的优化
- 减少层数: 仅使用2层编码器
- 缩小模型维度: d_model=64（标准Transformer常用512或更大）
- 增加Dropout: 0.3（防止过拟合）
- Pre-LayerNorm: 提升训练稳定性

### 4.5 参数规模
- 可训练参数: ~102,913
- 模型维度: 64
- 注意力头数: 4
- 编码器层数: 2
- 前馈维度: 256

### 4.6 适用场景
- 复杂的序列建模任务
- 需要捕捉全局依赖关系
- 对预测精度要求较高的场景

## 五、训练策略改进

### 5.1 优化器: AdamW
- 解耦权重衰减，比Adam更好的正则化效果
- 权重衰减系数: 1e-3

### 5.2 学习率调度: Cosine Annealing with Warmup
```
Warmup阶段 (前20个epoch): lr = base_lr × epoch / warmup_epochs
Cosine阶段: lr = min_lr + 0.5 × (base_lr - min_lr) × (1 + cos(π × progress))
```

### 5.3 混合损失函数
```
Loss = 0.7 × MSE + 0.3 × SmoothL1
```
- MSE: 对大误差敏感，优化整体精度
- SmoothL1: 对异常值更鲁棒，稳定训练

### 5.4 正则化策略
- Dropout: 0.3
- 权重衰减: 1e-3
- 梯度裁剪: max_norm=1.0
- Early Stopping: patience=50

## 六、数据处理改进

### 6.1 风向周期性编码
```python
sin(angle), cos(angle)  # 将8个离散风向编码为2维连续特征
```
- 保持方向的连续性（东和东北的距离比东和西小）

### 6.2 时间特征工程
- 归一化时间: t / max_t
- 二次特征: (t / max_t)²
- 周期特征: sin(2πt), cos(2πt)

### 6.3 交互特征
- 风速 × 归一化时间

### 6.4 RobustScaler
- 使用中位数和四分位距进行归一化
- 对异常值更鲁棒

## 七、模型对比总结

| 模型 | 参数量 | 核心特点 | 适用场景 |
|------|--------|----------|----------|
| ResidualMLP | ~102K | 残差连接、深层MLP | 逐点预测、快速训练 |
| CNN-BiLSTM | ~64K | 局部特征+双向时序 | 形状特征提取 |
| AttentionLSTM | ~241K | 时序建模+全局注意力 | 远程依赖建模 |
| ImprovedTransformer | ~103K | 纯注意力、并行计算 | 复杂全局依赖 |

## 八、使用建议

1. **小样本优先**: 在64个样本的情况下，建议优先尝试参数量较小的CNN-BiLSTM
2. **精度优先**: 如果追求最高精度，ImprovedTransformer通常表现最好
3. **速度优先**: ResidualMLP训练最快，适合快速实验
4. **集成使用**: 可以将多个模型的预测结果加权平均，进一步提升性能
